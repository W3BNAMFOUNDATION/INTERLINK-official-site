üß© Architecture Overview

Ingest Pipeline

Inputs from text, audio, PDFs, URLs, files, and system logs flow into the embedder, then into the local FAISS-compatible vector store.

Embedders

Use Ollama, DeepSeek, or Groq encoders‚Äîfully offline if desired‚Äîto generate high-quality embeddings with GPU acceleration.

Memory Shards

Data is persisted to zero-dependency shards: .jsonl for text chunks, .index for binary vectors, and .meta for timestamps and metadata.

All shards live at /workspace/memory/ for portable, local-only storage.

Retrieval & Recall

Semantic search spans every shard with cosine similarity, multi-query expansion, and contextual re-ranking to keep answers accurate and fast.
