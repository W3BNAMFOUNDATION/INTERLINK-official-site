Documentation

Your complete reference for using INTERLINK™ 2.0

1. Overview

INTERLINK™ is a unified AI workspace for agents, memory, knowledge engines, automation, files, and secure operations. This documentation provides technical and conceptual guidance for installation, usage, APIs, and architecture.

2. System Architecture

  • Agent Router — multi-model prompt routing.
  • Memory Engine — vector + JSONL + metadata portable storage.
  • Knowledge Engine — structuring & processing pipelines.
  • Automation Engine — triggers, workflows, and batch operations.
  • File Engine — safe, sandboxed file operations.
  • CLI Suite — local development tools.

3. Backend Modules

/workspace/
    memory/      → .jsonl, .index, .meta
    ingest/      → URL, PDF, TXT, YouTube
    filesystem/  → isolated file manager
    gibberlink/  → agent communication layer
    cli/         → local developer tools

4. Installation

Follow full instructions via the Install page, including the one-shot installer.

5. APIs

5.1 /api/send

Sends prompts to selected service profile (Ollama, DeepSeek, Groq, etc.).

5.2 /api/memory

Store, retrieve, and query long-term memory shards.

5.3 WebSocket Streaming

Real-time streaming channel for agent responses.

6. CLI Commands

  • interlink chat — talk to the router
  • interlink ingest — add data to the memory engine
  • interlink raw — direct model access

7. Troubleshooting

  • Check Python version (3.10 recommended).
  • Ensure service_profiles.yaml is correctly configured.
  • Run pip install -r requirements.txt
