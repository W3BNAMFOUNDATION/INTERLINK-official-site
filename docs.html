INTERLINK™ — Ingest Engine

Capture, process, and transform every source into structured intelligence.

The ingest engine is the input layer of the sovereign INTERLINK™ stack, turning documents, media, and live feeds into semantic chunks enriched with metadata and vector embeddings.

Built-in ingestion types

  • ✓ PDFs with multi-column reading and OCR fallback
  • ✓ URLs and webpages with link sanitization and clutter removal
  • ✓ YouTube transcripts with timestamped chunking and language selection
  • ✓ Local files: TXT, MD, HTML, RTF, CSV, JSON, and logs
  • ✓ Images with OCR, transcripts, and raw data streams
  • ✓ API feeds and offline, air-gapped ingestion

Outputs

  • • Semantic chunks optimized for retrieval
  • • Metadata-rich knowledge packets
  • • Structured memory entries
  • • Searchable vector embeddings

Autonomous jobs

Schedule ingestion runs, periodic scraping, batch PDF processing, and enrichment campaigns. Pipelines can be triggered manually, via agents, from the CLI, or fully offline.

INTERLINK™ — Secure Filesystem Engine

Hardened, sandboxed I/O for every agent, memory job, and automation pipeline.

The filesystem engine is the controlled access layer for the entire INTERLINK™ ecosystem, delivering deterministic paths, namespaced workspaces, and automatic cleanup across Windows, Linux, and Mac.

Core principles

  • ✓ Sandboxed operations with read/write isolation per agent
  • ✓ Deterministic, normalized file paths with traversal protection
  • ✓ Namespaced workspaces for ingest, memory, automation, and uploads
  • ✓ Secure temp directories with automatic expiration
  • ✓ Cross-platform compatibility for sovereign and offline deployments

Security model

  • • Path whitelisting to restrict access to approved directories
  • • File type enforcement to block unsafe executables
  • • Quotas to prevent runaway storage and oversized writes
  • • UTF-8 normalization and BOM removal on reads
  • • Atomic writes with built-in versioning for corruption-safe saves

Filesystem layout

Namespaced directories keep ingest, memory, and automation work isolated from one another.

/workspace/
  /ingest/
  /memory/
  /agents/
  /automation/
  /tmp/
  /uploads/

Built-in tools

  • ✓ Safe directory creation, listing, and removal
  • ✓ Automated temp cleanup and sandbox resets
  • ✓ MIME detection and upload validation
  • ✓ Workspace-scoped access policies for agents
  • ✓ API and CLI hooks for controlled file commands

Sample operations

Deterministic commands used by agents, ingest jobs, and automation scripts.

filesystem.write("workspace/notes/test.txt", "Hello World")
content = filesystem.read("workspace/notes/test.txt")
files = filesystem.list("workspace/notes/")
filesystem.reset()

Workspace quickstart

  1. Install the desktop app and sign in with your workspace key.
  2. Create your first project and select your preferred LLM stack (online/offline).
  3. Connect storage: choose local embeddings or your enterprise vector database.
  4. Import a PDF, URL, or repository into memory and start chatting with the context.

APIs

Call INTERLINK™ 2.0 from any stack. REST and WebSocket endpoints are available once the desktop service is running.

EndpointDescription
/api/v2/agentsCreate, list, and manage agent profiles.
/api/v2/memory/searchQuery embeddings with hybrid semantic/keyword filters.
